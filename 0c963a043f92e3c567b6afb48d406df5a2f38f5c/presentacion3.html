Importar Datos de Fuentes Externas

  • Rstudio tiene una opcin para incorporar bases de datos de manera ms fcil

  • Existen muchos tipos de datos (.csv, .sav, .Rds, .xlsx, etc.)
  • En este caso, vamos a ver el CSV (archivo de texto separado por comas)
  • Pero puede estar separado por distintos caracteres (“;”, “,”, “\t o TAB” )
  • Los elementos que definen a una importacin de un csv que hay que tener en consideracin son los siguientes:
    • Elemento separador de celdas
    • Norma de codificacin de texto (UTF-8, Latin1,etc.)
    • Punto de partida de recogida de datos
    • Primera fila como nombre de las columnas?
    • Separador de Decimales

  • Cargue la base de datos enviada por correo electrnico (si no le lleg, bjela por aqu.

  • En el caso de la moda necesitaremos aplicar una funcin, que tomar todos los valores nicos de un determinado vector, y que obtendr el que posee la mayor cantidad de observaciones del vector que calzan con el valor nico.
Mode <- function(x) {
  ux <- unique(x)
  ux[which.max(tabulate(match(x, ux)))]
}
gastos_casa_csv_2<-cbind(Gastos_casa_csv, monto_en_miles=round(Gastos_casa_csv$monto/1000,0))
Mode(gastos_casa_csv_2$monto_en_miles)

Configuracin de Datos

  • Bajar BASE DE DATOS

  • Los elementos que definen a una importacin de un excel (*.xlsx) que hay que tener en consideracin son los siguientes:

    -Nombre que se le asignar a base de datos

    -Nombre de la hoja de excel que se quiere importar a R

    -Rango de datos (para evitar lo que ocurri con los gastos de la casa)

    -saltar algunas filas hasta llegar al nombre de las columnas (generalmente til si las primeras filas no son de utilidad para diferenciar los datos, como el nombre del autor, la fecha,o nombres de un conjunto de filas, etc.)

    -NA (cmo se tratan los valores perdidos)

    -Checklist si se considera la primera columna como nombres de columna (si utiliz el argumento skip, este ltimo antecede esta opcin) -Si quiere abrir un visor de datos (View()) - ojo con las maysculas
